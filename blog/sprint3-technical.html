An analogy to describe the differences between HTML and CSS

Imagine we are building a house, its generally good practice to use a blueprint. This allows us to imagine the layout of the house before we build - how it's going too look once it's finished:

HTML is similar to laying down concrete, we pour cement into the area in which we have already marked out. We need make sure we have a stable foundation for the house to sit on. Then we begin building walls, leaving space for plumbing and the electricity further down the track. Once we're satisfied the house is stable and the roof is on - that there are no bedrooms missing, we then begin to furnish and decorate using CSS.

CSS is when we begin painting the house, choosing the tiles for the bathroom. Whether or not we want marble countertops in the kitchen. We generally go for consistent style of decor - but there's also nothing stopping you from putting a shag carpet in the garage if it will delight any visitors.

JavaScript is where we start connecting all the wiring, gas and water supplies. Making sure the taps have running water and that the light switches work when they're turned on.

Explain control flow and loops using an example process from everyday life

Loops and Control flow are comparable to cooking a set meal.

The general flow is to eat the appetizer, the main and then dessert. When we cook a lasagna, we first preheat the oven on before placing the raw ingredients inside to bake. We cannot cook the lasagna if the oven hasn't been turned on beforehand.

Our eating routine is generally a loop, one that starts with breakfast, lunch and dinner. Sometimes we skip breakfast, and if we have already eaten lunch that day then our next meal is dinner.

The Document Object Model is the thing that allows us to interact with the interweb. We view the document through a window. What we are seeing is a collection of different text documents that contain instructions.

Those individual documents contain the instructions for how the page will look and behave.
